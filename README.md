# Osoul Pro | أصول برو

نظام ويب لإدارة وتتبع محفظة العملات الرقمية، مبني كواجهة أمامية (Frontend) بدون خادم خلفي، مع دعم PWA والعمل دون اتصال جزئيًا.

## ما الذي يقدمه المشروع؟

- إدارة العملات والصفقات (شراء/بيع) وحساب متوسط التكلفة والأرباح والخسائر.
- جلب أسعار السوق من عدة مصادر مع آلية احتياطية (Fallback) وكاش محلي للأسعار.
- حساب رسوم Spot حسب المنصة والدولة (Maker/Taker) عبر إعدادات مخصصة.
- بطاقات **Market Insights** (Market Cap / Fear & Greed / Altcoin Season / أخبار كريبتو).
- نسخة احتياطية واستيراد للبيانات بصيغة JSON.
- وضع خصوصية مالي لإخفاء الأرقام الحساسة داخل الواجهة.
- دعم الثيم الفاتح/الداكن، إشعارات داخلية، وتجربة PWA.

## التشغيل المحلي

يفضل تشغيل المشروع عبر خادم محلي حتى تعمل ميزات PWA وService Worker بشكل صحيح.

### المتطلبات

- متصفح حديث.
- اتصال إنترنت (لجلب الأسعار والبيانات الخارجية).

### خطوات التشغيل

1. افتح المجلد:

```powershell
cd C:\Users\user\Desktop\Osoul-Pro
```

2. شغّل خادمًا محليًا (أحد الخيارين):

```powershell
python -m http.server 8080
```

أو:

```powershell
npx serve .
```

3. افتح التطبيق في المتصفح:

```text
http://localhost:8080
```

> الصفحة الجذرية `index.html` تقوم بتحويلك تلقائيًا إلى `pages/index.html`.

## بنية المشروع

- `pages/index.html`: الواجهة الرئيسية للتطبيق.
- `assets/js/script.js`: منطق التطبيق الأساسي (الأسعار، الحسابات، الواجهة، التحديثات).
- `data/exchanges.spot.js`: بيانات المنصات والدول ورسوم Spot.
- `services/feeCalculator.js`: خدمة حساب الرسوم.
- `ui/`: وحدات واجهة إضافية (الإعدادات، الجداول، نافذة الإعداد الأولي).
- `service-worker.js`: التخزين المؤقت ودعم العمل دون اتصال.
- `manifest.json`: إعدادات تطبيق الويب التقدمي (PWA).
- `pages/privacy.html`, `pages/terms.html`, `pages/disclaimer.html`: الصفحات القانونية.

## الخصوصية والبيانات

- يعتمد التطبيق على `localStorage` لحفظ بياناتك وإعداداتك محليًا في المتصفح.
- لا توجد قاعدة بيانات أو Backend داخل هذا المستودع.
- هذا المشروع أداة متابعة وإدارة، وليس نصيحة استثمارية.

## تحديث الرسوم والإصدارات

- تحديث رسوم المنصات: عدّل ملف `data/exchanges.spot.js`.
- تحديث ملاحظات الإصدار المنبثقة: عدّل `update_popup.json`.
- تحديث كاش PWA: حدّث رقم الإصدار في `service-worker.js` عند كل إصدار جديد.
- عند كل رفع Frontend: حدّث رقم `v=` لملف `assets/js/script.js` في `pages/index.html` و`service-worker.js` لتفادي تحميل نسخة قديمة من الكاش.
